📝 논문

✍️ 문제 제기

  • 클라우드 제공업체들은 잉여 자원 활용을 극대화하기 위해 스팟 인스턴스를 할인된 가격으로 제공하지만, 갑작스러운 중단 위험이 있음
  • 제공업체들은 스팟 인스턴스의 안정적 사용을 돕기 위해 다양한 도구를 제공
  • 특히 AWS는 Spot Placement Score(SPS)를, Microsoft Azure는 Spot Placement Recommender와 같은 가용성 데이터를 제공하고 이를 이용한 스팟 인스턴스 가용성 개선 연구가 진행됨
  • 하지만 기존 가용성 개선 연구들은 단일 노드 환경에만 초점을 맞추고 있어, 최근 증가하고 있는 다중 노드 환경에서의 안정성 문제를 해결하지 못함

🚧 핵심 기여

  • 쿼리 제한을 극복하는 Uniform Spacing Query Sampling 기법으로 AWS의 스팟 다중 노드 가용성 데이터 쿼리
  • 쿼리한 스팟 다중 노드 가용성 데이터셋을 분석하여 비용과 안정성의 균형을 맞춘 최적의 인스턴스를 추천하는 알고리즘 제안
  • 제안할 알고리즘을 바탕으로 스팟 인스턴스 추천 시스템 Sandevistan 구현
  • 제안된 시스템 Sandevistan은 AWS SpotFleet 대비 20.8% 더 나은 안정성과 26.3% 더 높은 비용 절감을 달성

🖥️ Sandevistan 시스템 구성도

👨🏻‍💻담당 업무

  • Athena에서 멀티노드 가용성 데이터를 바로 점수화하는 SQL문 작성
    • 기존 쿼리한 다중 노드 가용성 데이터로는 스팟 인스턴스의 가용성을 나타내기 어려움
    • 또한 Uniform Spacing Query Sampling 기법으로 빈 데이터 존재
    • 이에 빈 데이터를 ForwardFill로 채우고 이를 점수화하는 쿼리문 작성
  • 스팟 인스턴스 추천 알고리즘 제안
    • 최적의 스팟 인스턴스를 선택하기 위해 가용성과 비용을 모두 고려
    • 스팟 가격을 정규화하여 가격점수를 제안
    • Athena에서 쿼리한 가용성 데이터를 이용하여 안정성 점수를 제안
    • 가격점수와 가용성 점수를 제안 후 둘을 결합하여 최종 점수 계산
    • 최종 점수를 최대화 하는 다목적 최적화 문제를 이용하여 최적의 스팟 인스턴스 선택
  • 스팟 인스턴스 중단 실험 진행
    • 기존 스팟 인스턴스 중단 데이터를 수집하기 위해서는 많은 비용이 발생
    • 실험 비용을 줄이기 위해 주기적으로 스팟 인스턴스가 할당 가능한지 확인하는 방법을 제안
    • Terraform을 이용하여 주기적 요청으로 스팟 인스턴스 중단 데이터를 수집할 수 있는 코드 작성
  • 통계모델을 이용한 중단 데이터 분석
    • 스팟 인스턴스 중단 실험을 바탕으로 생존 분석에 사용되는 통계모델인 Kaplan-Meier Estimator와 Cox Proportional Hazards Model를 사용 제안한 가용성 점수와 생존 확률의 상관관계를 밝힘
      • 분석 결과 가용성 점수가 높을수록 인스턴스의 생존 시간이 증가하고 중단 확률이 감소함을 정량적으로 증명
      • 가용성 점수가 0점에서 100점으로 증가할 때 스팟 인스턴스 생존 시간 중앙값이 8.6시간 증가하고, 중단 확률이 62.1% 감소함을 확인
  • 추천 시스템과 SpotFleet비교
    • 추천 시스템의 효율성을 증명하기 위해 기존 스팟 인스턴스 추천 서비스인 AWS SpotFleet과 비교함
    • 그 결과 제안한 Sandevistan 시스템은 SpotFleet대비 가용성을 20%p향상시켰고 비용을 25%p 절감할 수 있었음
  • 결과 시각화
    • 위 내용을 기반으로 python의 matplotlib과 numpy라이브러리를 이용하여 분포, 변화 패턴, 변화 빈도 등 시각화 및 분석
